# Почему навыков недостаточно: принципы и генеративная иерархия

> **Source-of-truth:** [PD.FORM.047](https://github.com/TserenTserenov/PACK-personal/blob/main/pack/personal-development/02-domain-entities/formalizations/PD.FORM.047-generative-principles-hierarchy.md)

## Главная идея

Навыки (skills) — необходимый, но **недостаточный** уровень мастерства. Они находятся на самом нижнем уровне четырёхуровневой иерархии принципов и **порождаются** принципами, а не заменяют их.

## Четыре уровня

| Уровень | Что | Зачем агенту | Без него |
|---------|-----|-------------|----------|
| **Level 0: ZP** | Нулевые принципы (трансдисциплины) | Способность разбираться в ЛЮБОМ новом домене | Каждая смена области = начало с нуля |
| **Level 1: FPF** | Первые принципы (рамка корректности) | Точный язык: Role ≠ Method ≠ Work ≠ Capability | Всё называется «процесс» — путаница |
| **Level 2: Pack** | Вторые принципы (доменное знание) | Source-of-truth: различения, методы, failure modes | Tribal knowledge, каждый проект заново |
| **Level 3: DS** | Навыки, программы, инструменты | Конкретное исполнение | Нечего делать |

**Fallback chain:** DS → Pack → SPF → FPF → ZP. Если на уровне DS непонятно — поднимаешься к Pack. Если Pack не помогает — к FPF. И так до ZP.

## Четыре аргумента: почему навыков недостаточно

### 1. Навык процедурен, а не стратегичен

Навык — автоматизированная последовательность операций. Он отвечает на вопрос **«как выполнить ЭТУ процедуру»**. Но вопрос **«КАКУЮ процедуру выбрать»** — это знание, порождённое принципами.

Аналогия из ML (MIM.D.007): принцип — это inductive bias архитектуры, навык — обученные веса. Без правильного bias никакие данные не помогут.

### 2. Навыки не переносятся между доменами

Навык программиста бесполезен в хирургии. Навык бухгалтера бесполезен в дизайне. Дисциплинарные навыки привязаны к контексту.

Нулевые принципы (ZP) и трансдисциплины переносимы по определению: онтология, логика, системное мышление работают везде. Тот, кто владеет трансдисциплинами, осваивает новый домен за недели, а не за годы.

### 3. «Ведро навыков» ≠ интегрированная способность

Команда, где каждый умеет делать A, B, C по отдельности — не то же, что команда, которая освоила интегрированный рабочий процесс A+B+C. Эмерджентная ценность от интеграции невидима, если смотреть только на навыки.

Мастерство = навыки + координация + суждение + выбор метода. Последние три — не навыки, а знание.

### 4. Навыки не порождают новых навыков

Принципы — генеративны: из них можно вывести новые методы для новых ситуаций. Навыки — исполнительны: работают только в тренированных ситуациях.

При технологическом сдвиге (ИИ замещает функции, появляются новые профессии) агент с принципами создаёт новые роли. Агент только с навыками ищет курсы переподготовки.

## Генеративная цепочка

```
ZP (думать)  →  FPF (описывать)  →  Pack (знать)  →  DS (делать)
     ↓                ↓                   ↓                ↓
 Перенос         Корректность       Доменный язык      Навыки
 между           описаний           и методы            и программы
 доменами
```

Каждый уровень **порождает** следующий. Без верхних уровней нижние не возникают или возникают медленно и хрупко.

## Что это значит для пользователя IWE

IWE построена по этой иерархии:

| Компонент IWE | Уровень принципов |
|---------------|-------------------|
| Методология (FPF/SPF, протоколы) | Level 0-1 |
| Экзокортекс (CLAUDE.md, memory/) | Level 1-2 |
| Роли ИИ (Стратег, Экстрактор) | Level 2-3 |
| Рабочее пространство (DS-strategy/) | Level 3 |
| Инструменты (Git, MCP, бот) | Level 3 |

Пользователь IWE не просто получает инструменты (Level 3). Он получает **методологию** (Level 0-1) и **доменное знание** (Level 2), которые позволяют порождать новые навыки и адаптироваться к изменениям.

## Связь с форматами развития (МИМ)

| Уровень | Что осваивается | Формат |
|---------|----------------|--------|
| ZP/FPF | Принципы | Семинар (знакомство) → Практикум (автоматизация) |
| Pack | Доменное знание | Семинар + Практикум |
| DS | Навыки | Практикум → Резидентура (рабочий результат) |
