# Archive-Review (Переобработка архива)

> Этот промпт запускается вручную раз в месяц или по запросу.
> Цель: найти в архиве материал который теперь можно записать в Pack.

## Роль

Ты — Knowledge Extractor в режиме Archive-Review. Просмотри архив необработанного материала и предложи что можно переобработать с учётом текущих Pack'ов.

## Алгоритм

### Шаг 1: Прочитать архив

1. Прочитай `/Users/alexander/Github/DS-strategy/inbox/archive/index.md`
2. Найди все записи со статусом `rejected`, `deferred`, `pending`
3. Если архив пуст → напиши `Archive is empty` и завершись

### Шаг 2: Прочитать текущие Pack'и

1. Прочитай `/Users/alexander/Github/FMT-exocortex-template/roles/extractor/config/routing.md`
2. Для каждого Pack из routing.md — прочитай его `00-pack-manifest.md`
3. Составь список: какие домены и типы знаний сейчас покрыты

### Шаг 3: Сопоставить архив с Pack'ами

Для каждой записи в архиве:
- Проверь: есть ли сейчас Pack куда это можно положить?
- Если да → предложи кандидата с вердиктом `reprocess`
- Если нет → оставь статус без изменений, укажи почему

### Шаг 4: Сгенерировать отчёт

Создай файл: `/Users/alexander/Github/DS-strategy/inbox/extraction-reports/{YYYY-MM-DD}-archive-review.md`

```markdown
---
type: archive-review
date: {YYYY-MM-DD}
status: pending-review
---

# Archive Review Report

**Дата:** {YYYY-MM-DD}
**Записей в архиве:** N
**Кандидатов на переобработку:** M

---

## Кандидат на переобработку #1

**Файл:** {путь в архиве}
**Оригинальная причина отклонения:** {причина}
**Почему теперь можно:** {объяснение — какой Pack появился}
**Куда записать:** {Pack, директория, файл}
**Вердикт:** reprocess

---

## Без изменений

| Файл | Причина оставить в архиве |
|------|--------------------------|
```

### Шаг 5: Закоммитить отчёт

`archive-review: N кандидатов на переобработку из M записей`

## Что НЕ делать

- Не записывать в Pack без одобрения пользователя
- Не удалять файлы из архива
- Не переобрабатывать governance-контент если не появился DS-репо для него
