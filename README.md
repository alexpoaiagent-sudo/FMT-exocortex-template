# Персональный экзокортекс

> Рабочее пространство для интеллектуальной работы и развития с ИИ-агентами.

Ты проходишь курсы. Бот помогает каждый день. Но знания остаются в голове, планы — в заметках, а контекст теряется между сессиями.

**Экзокортекс** — это твоё личное рабочее пространство, где Claude помогает планировать неделю, фиксировать знания и строить собственную базу. Он подключён к платформе: те же знания, тот же бот, но теперь ещё и личный стратег.

## Путь на платформе

Платформа растёт вместе с тобой. Каждый тир разблокирует новую функциональность.

> **Жирный шрифт** — уже работает. Обычный шрифт — в плане развития (номер этапа в скобках).

```
T1: Старт  →  T2: Изучение  →  T3: Персонализация  →  T4: Созидание  →  T5: Администрирование
Бесплатно      БР                БР+                     БР++              Владелец
```

| | T1: Старт | T2: Изучение | T3: Персонализация | T4: Созидание | T5: Администрирование |
|---|---|---|---|---|---|
| **Подписка** | **Бесплатно** (бот: Telegram Stars) | **БР** | **БР+** | **БР++** | Владелец |
| **Бот** | **Марафон, Лента, Консультации** | **Ограниченный функционал** | **Продвинутые ассистенты по руководствам** | — | — |
| **Контент** | — | **Руководства, Программы** | то же | то же | то же |
| **Сообщество** | — | **Клуб, Чаты** | то же | то же | то же |
| **ДЗ-чекер** | — | **Проверка домашних заданий** | то же | то же | то же |
| **ЦД** | **Ограниченный** | **Полный** | **Полный + персональное развитие** | **Полный** | **Полный** |
| **Персонализация** | — | — | **Персональные ответы** | **+ личные правила, рабочее пространство** | то же |
| **Агенты** | — | — | — | **Claude Code + Стратег + Экстрактор + Синхронизатор** | **+ все агенты** |
| **Свои БД** | — | — | — | **Подключение к своим базам данных** | то же |
| **Аналитика** | Частичная (Э2) | По активным действиям (Э2) | По рабочему процессу (Э3) | Полная по себе (Э3) | По всем участникам и сообществу (Э4) |
| | Марафон (Э1) | | | Standard/Personal split (Э1) | |
| | | | Рефайнмент в консультации (Э3) | Документация Экстрактора (Э2) | |
| | | | | Документация Синхронизатора (Э2) | |
| | | | | Авто-детекция компонентов (Э4) | |

> Названия тарифов БР+/БР++ — рабочие. Финальные названия, обоснования и стоимость — в разработке.

## Что даёт T4: Созидание

После установки у тебя будет работающая система из 4 компонентов:

| Компонент | Что делает |
|-----------|-----------|
| **CLAUDE.md** | Правила для Claude Code: как открывать сессию, как фиксировать знания, как закрывать. Claude помнит контекст между сессиями |
| **memory/** | Оперативная память: текущие задачи, различения, чеклисты. Claude читает их в начале каждой сессии |
| **Стратег** | Агент, который запускается автоматически: утренний план дня, вечернее ревью, недельная сессия стратегирования |
| **DS-strategy/** | Твой стратегический хаб: планы недель, отчёты, неудовлетворённости, входящие заметки |

**Как это выглядит на практике:**

- Утром — получаешь план дня в Telegram (Стратег составил его ночью)
- Открываешь VS Code → `claude` → Claude знает, что в плане, и предлагает начать с приоритетного
- Работаешь — Claude фиксирует знания по ходу (Capture-to-Pack)
- Закрываешь сессию — результат зафиксирован, план обновлён
- В понедельник — Стратег готовит черновик недельного плана, вы обсуждаете его на сессии стратегирования

## Быстрый старт (T4: Созидание)

### Требования

- macOS (launchd для автоматического запуска агентов)
- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) (CLI)
- Git + GitHub аккаунт
- [GitHub CLI](https://cli.github.com/) (`gh`)

### 1. Fork

```bash
gh repo fork TserenTserenov/FMT-exocortex-template --clone
```

### 2. Автоматическая настройка

```bash
gh repo clone TserenTserenov/DS-exocortex-setup-agent
cd DS-exocortex-setup-agent
bash setup.sh
```

Скрипт спросит: GitHub username, рабочую директорию, путь к Claude CLI и таймзону. Всё остальное — автоматически.

<details>
<summary>Ручная настройка (если setup.sh не подходит)</summary>

1. Замените `{{GITHUB_USER}}` на ваш GitHub username во всех файлах
2. Замените `{{WORKSPACE_DIR}}` на путь к рабочей директории (напр. `~/Github`)
3. Замените `{{TIMEZONE_HOUR}}` на час запуска стратега в UTC (напр. `4` для 7:00 MSK)
4. Замените `{{TIMEZONE_DESC}}` на описание времени (напр. `7:00 MSK`)
5. Замените `{{CLAUDE_PATH}}` на путь к Claude CLI (напр. `/opt/homebrew/bin/claude`)
6. Установите launchd-агентов: `cd strategist-agent && bash install.sh`
7. Скопируйте `memory/` в `~/.claude/projects/.../memory/`
8. Скопируйте `CLAUDE.md` в корень рабочей директории

</details>

### 3. Первая сессия

```bash
cd ~/Github    # или ваша рабочая директория
claude
```

Попросите Claude провести первую стратегическую сессию: определить цели, зафиксировать неудовлетворённости, составить план первой недели.

## Рост внутри T4: Созидание

Базовый набор (Claude Code + Стратег + DS-strategy) работает сразу. Когда дорастёшь — добавляй опциональные компоненты:

| Компонент | Когда добавлять | Что даёт |
|-----------|----------------|----------|
| **Экстрактор** | Создал первый Pack (область знаний) | Автоматическое извлечение знаний из сессий в Pack |
| **Синхронизатор** | Появилось 3+ репозитория | Кросс-репо синхронизация, code-scan, автоматические проекции |

Принцип: минимальная сложность на старте, усложнение по мере роста.

## План развития

### Этап 1: Публичный запуск (W09)

| Что делаем | Тир | Пользователь получает |
|-----------|-----|----------------------|
| Марафон через бота (пошаговый курс) | T1: Старт | Учится внутри бота: теория → практика → рефлексия |
| README для широкой аудитории | T4: Созидание | Понимает, зачем и как поставить экзокортекс |
| Чистый MEMORY.md skeleton | T4: Созидание | Стартует с пустой таблицей задач |
| Standard/Personal split memory/ | T4: Созидание | Обновления из upstream не ломают личные файлы |
| Smoke test (fork → setup → validate) | T4: Созидание | Гарантия: setup работает для нового пользователя |

### Этап 2: Аналитика + компоненты T4 (W10-W11)

| Что делаем | Тир | Пользователь получает |
|-----------|-----|----------------------|
| Частичная аналитика (бот) | T1: Старт | Видит свой прогресс: активность, streak |
| Аналитика по активным действиям | T2: Изучение | Видит: сколько руководств прошёл, ДЗ сдано, прогресс |
| Документация self-install Экстрактора | T4: Созидание | Создал Pack → добавь Экстрактор |
| Документация self-install Синхронизатора | T4: Созидание | 3+ репо → добавь Синхронизатор |
| setup.sh `--with-extractor` / `--with-synchronizer` | T4: Созидание | Одна команда вместо ручной настройки |

### Этап 3: Персонализация + аналитика процесса (W11-W12)

| Что делаем | Тир | Пользователь получает |
|-----------|-----|----------------------|
| Рефайнмент в консультации (уточняющие вопросы) | T3: Персонализация | Если ответ неполный → бот спрашивает «хочешь углубиться?» |
| Аналитика по рабочему процессу | T3: Персонализация | Видит: как учится, что работает, где застревает |
| Полная аналитика по себе | T4: Созидание | Видит: РП, captures, знания, рост компетенций |

### Этап 4: Авто-детекция + платформа (W12+)

| Что делаем | Тир | Пользователь получает |
|-----------|-----|----------------------|
| Auto-detect: создал Pack → предложить Экстрактор | T4: Созидание | Система сама направляет рост |
| Auto-detect: ≥3 репо → предложить Синхронизатор | T4: Созидание | Система предлагает следующий компонент |
| Аналитика по всем участникам и сообществу | T5: Администрирование | Видит: активность сообщества, тренды, здоровье платформы |

## Обновления из upstream

```bash
git remote add upstream https://github.com/TserenTserenov/FMT-exocortex-template.git
git fetch upstream
git merge upstream/main
```

Обновления протоколов и промптов приходят из upstream. Твои личные данные (планы, memory, стратегия) не затрагиваются.

## Структура

```
экзокортекс/
├── CLAUDE.md                       # Правила для Claude Code (протоколы, архитектура)
├── memory/                         # Оперативная память (7 файлов, ≤100 строк каждый)
│   ├── MEMORY.md                   # Задачи недели + навигация (авто-загрузка)
│   ├── fpf-reference.md            # Первые принципы (FPF)
│   ├── hard-distinctions.md        # Жёсткие различения
│   ├── checklists.md               # Чеклисты
│   ├── sota-reference.md           # SOTA-практики
│   ├── repo-type-rules.md          # Правила по типам репозиториев
│   └── claude-md-maintenance.md    # Обслуживание CLAUDE.md
├── strategist-agent/               # Агент-стратег
│   ├── prompts/                    # Промпты (session-prep, day-plan, week-review...)
│   ├── scripts/                    # Скрипты запуска
│   └── launchd/                    # Расписание (macOS)
└── my-strategy/                    # Стратегический хаб (→ DS-strategy после setup)
    ├── current/                    # Текущие планы и отчёты
    ├── archive/                    # Архив
    ├── inbox/                      # Входящие заметки
    └── docs/                       # Стратегия, неудовлетворённости
```

## Принципы

1. **Platform-space vs User-space** — протоколы и промпты (platform) обновляются из upstream. Твои планы, память и стратегия (user) — только у тебя
2. **3-слойная память** — MEMORY.md (всегда в контексте, ≤100 строк) → CLAUDE.md (правила, ≤300 строк) → memory/*.md (справочники, по запросу)
3. **Capture-to-Pack** — знания фиксируются по ходу работы, не теряются
4. **WP Gate** — нетривиальная работа начинается с проверки плана. Нет в плане = не делаем
5. **Open → Work → Close** — каждая сессия: открытие (что делаем) → работа (с фиксацией знаний) → закрытие (результат зафиксирован)
